# Portfolio 3.0 — Identity, Motion & Project Refresh

**Date:** 2026-06-16
**Owner:** Christian Raro
**Status:** Approved (design) → ready for implementation plan

## Goal

Add more visual identity and personality to the existing Next.js 14 portfolio
without changing its architecture: an editorial typography system, tasteful
motion add-ons, rewritten About copy, a curated/up-to-date project list, and —
importantly — project cards that **display a mockup screenshot** (today the
screenshot only appears on hover). Capture fresh screenshots for every live
project and wire them in responsively.

## Non-Goals (YAGNI)

- No framework, routing, or data-model rewrite. Stay on Next.js 14 + Tailwind +
  Framer Motion.
- No CMS / backend changes. Project data stays in `lib/data.ts`.
- No new pages beyond the existing `/`, `/projects`, `/tech-stack`.
- No mobile-app screenshots / device frames — desktop site mockups only.

## Current State (as explored)

- Single-page bento layout in `app/page.tsx`, plus `/projects` and `/tech-stack`.
- Typography: **Plus Jakarta Sans only** (`app/layout.tsx`, `app/globals.css`,
  `tailwind.config.js`) — one font for everything, no display personality.
- Motion: minimal repeated inline `fade-in-up` + stagger variants across sections.
- **Project cards do not render `project.image`** — it is used only as the
  `fallbackImage` for the hover `LinkPreview` (microlink API). Cards in
  `components/sections/works-section.tsx` and `app/projects/page.tsx` are
  text-only.
- Color system: black/white + red accent (`#FF2D55`), oklch tokens, light/dark.
- `scripts/capture-project-screenshots.mjs` references **puppeteer, which is not
  installed**, and only targets 4 old URLs. It will be replaced/superseded by the
  `gstack` headless-browser capture flow.

## Design

### 1. Typography — Modern editorial (serif display + sans body)

- Add **Fraunces** (Google font, variable, optical-size serif) via
  `next/font/google` in `app/layout.tsx`, exposed as CSS var `--font-display`.
- Keep **Plus Jakarta Sans** for body/UI.
- Wire `--font-display` into `tailwind.config.js` `fontFamily.display`, and into
  `--font-display` usage in `globals.css`.
- Apply `font-display` to: hero name (`h1`), all section `h2` headings, and the
  `/projects` + `/tech-stack` page titles.
- Add a **fluid heading scale** using `clamp()` (utility classes or `globals.css`
  component layer) so editorial headings scale from mobile → desktop.
- Optional light touch: a monospace/uppercase **eyebrow label** (kicker) above
  sections (e.g. small tracked-out caps). Implemented as a tiny reusable class;
  trivially removable. (Use existing `--font-mono` token; no new font required.)

### 2. Motion add-ons (tasteful, accessible)

- New reusable client component `components/ui/reveal.tsx`:
  - `<Reveal>` — scroll-triggered fade/slide-in wrapper (replaces repeated inline
    variants), with optional `delay`.
  - A stagger-container variant for grids.
- Apply across sections to reduce duplication and unify timing.
- **Hero:** word-by-word reveal of the name in the display serif; soft fade for
  the supporting lines (extends current entrance, doesn't replace it wholesale).
- **Cards:** hover **lift + shadow** and **image zoom** within the thumbnail
  (`group-hover:scale-105` on the image, `overflow-hidden` container).
- **Links/CTAs:** animated underline / subtle hover + press states.
- **Accessibility:** all motion gated by `prefers-reduced-motion` (extend the
  existing media query in `globals.css`; `Reveal` checks the reduced-motion
  setting and renders static when set).

### 4. Project cards → visible responsive thumbnails

Redesign the card markup used in **both** `components/sections/works-section.tsx`
and `app/projects/page.tsx` (extract a shared `ProjectCard` if practical, or keep
them parallel):

- **Mockup image on top**, fixed aspect ratio (16:10), `next/image` with
  `object-cover object-top`, `overflow-hidden`, rounded top corners.
- Below image: title (display font), description (clamped), tech tags, domain.
- Keep the hover **`LinkPreview`** as a live full-page enhancement on top of the
  now-visible static thumbnail.
- Responsive grid: 1 col (mobile) → 2 (sm/md) → 3–4 (lg) as appropriate per page.
- Graceful fallback if an image is missing (neutral placeholder, no broken img).

### 6. Screenshots (recapture all live projects)

- Use the **`gstack`** headless browser to capture consistent desktop mockups
  (e.g. 1440×900 viewport, above-the-fold) for every project with a live URL.
- Save to `public/assets/images/projects/` using existing filename conventions
  (overwrite stale `.webp`/keep paths referenced by `lib/data.ts`).
- Update `lib/data.ts` `image` paths to match captured files where needed.
- Projects without a live URL: none remain after removing ARWay/Muramart — every
  retained project has a live URL.
- The stale `scripts/capture-project-screenshots.mjs` (broken puppeteer dep) is
  removed or updated to reflect the real flow. Decision: remove it to avoid a
  broken, misleading script; capture is done via `gstack`.
